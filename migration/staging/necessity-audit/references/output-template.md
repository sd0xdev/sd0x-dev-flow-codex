# Output template — Markdown report

Generated by `scripts/skills/necessity-audit/report.js::buildMarkdown` from `consolidate.js` output. The structure below is normative; deviations break hook parsing (`hooks/post-tool-review-state.sh:641` looks for the header + tail sentinel).

## Header (hook-parseable)

```markdown
## Document Review

**[OVERRIDE: feasibility included]**   <- each `banners[]` entry becomes its own bold line; omitted if empty
**[PREFLIGHT SKIPPED]**

**Target**: `<relative_path>`
**Feature**: `<feature_key>` (greenfield: <true|false>)
**Depth**: <brief|normal|deep> · **Preflight**: <advisory|skipped>
**Schema**: v1

> <warnings[0]>   <- each `warnings[]` entry rendered as a blockquote line; omitted if empty
```

## Sections (emitted in this exact order)

| # | Heading | Source |
|---|---------|--------|
| 1 | `### Dimension Overview` | `dimensions[1..6]` severity table — `Clean` / `Low` / `Med` / `High` / `Skipped` for inactive dims |
| 2 | `### Classification` with three subheadings: `#### Keep (N items)`, `#### Review (N items)`, `#### Cut (N items — unless overridden)` | `elements[]` grouped by `final` verdict. Review/Cut rows include Claude verdict, Codex verdict, evidence location, override indicator |
| 3 | `### Debate` | `debate.threadId`, `debate.rounds`, `equilibrium_reached`, compacted `conclusion`, `skill_invocation=codex-brainstorm` |
| 4 | `### Deterministic Checks (NFR-10)` | `deterministic_checks` object — 6 booleans rendered ✅/❌ (includes `conclusion_references_rounds` per SKILL.md Rule #4) |
| 5 | `**Under-covered dimensions**: ...` | Only emitted if `under_covered_dimensions[]` non-empty |
| 6 | `### Narrative` | Only emitted if `narrative[]` non-empty. Each line as a bullet. `⚠️ Need Human` / `⛔` / `ℹ️` prefixes come from the consolidator |
| 7 | `### Suggested Next` | Only emitted if `suggested_next[]` non-empty |
| 8 | `### Gate` | Single line: `✅ Mergeable` OR `⛔ Needs revision` — nothing else |

The gate value is ALWAYS one of those two — narrative may contain `⚠️ Need Human` lines but the sentinel stays binary for hook compatibility (see `hooks/stop-guard.sh` + `post-tool-review-state.sh`).
