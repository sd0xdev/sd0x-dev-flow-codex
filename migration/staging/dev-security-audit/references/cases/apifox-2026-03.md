# Apifox Supply Chain Attack (March 2026)

## Summary

Apifox CDN served a poisoned JavaScript file (`apifox-app-event-tracking.min.js`, 77KB vs normal 34KB) between 2026-03-04 and 2026-03-22. The malicious code had full Node.js RCE capability via the Electron app's unsandboxed environment. It collected machine fingerprints, stole credentials (SSH keys, Git tokens, shell history, K8s configs), and polled a C2 server every 30min-3hr with varying payloads. All three platforms (macOS, Windows, Linux) were affected.

## Appendix: Attack Chain Summary

The malware is a flexible RCE platform. The C2 can deliver completely different payloads on every polling cycle (30min-3hr random interval).

**Stage-1 (Loader, 344 bytes RSA encrypted)**: Dynamically creates a `<script>` tag to load `hxxps://apifox[.]it[.]com/<random-8-hex>.js`. URL is one-time-use. The `af_uuid` header is hardcoded into Stage-2 for client tracking. Script self-removes from DOM.

**Stage-2 v1 (`collectPreInformations`, ~3400 bytes)**: Recursively reads `~/.ssh/*`, shell histories, `~/.git-credentials`. Runs `ps aux` / `tasklist`. Exfil: JSON -> Gzip -> AES-256-GCM -> Base64 -> POST `/event/0/log`.

**Stage-2 v2 (`collectAddInformations`, ~4400 bytes)**: Adds `~/.zshrc`, `~/.npmrc`, `~/.kube/*`, `~/.subversion/*`, directory tree (depth 1-2). Windows scans D:\E:\F:\. Exfils to `/event/2/log`.

**Obfuscation (entry file)**: 7 layers — string array rotation (offset 275), Base64+RC4 dual decryption, proxy functions, hex arithmetic, control flow flattening, dead code, anti-debug traps. Stage-2 payloads were cleartext with Chinese comments (OPSEC inconsistency suggesting different authors for entry vs payload).

**C2 Server**: nginx/1.28.2 + Express. Random 8-hex Stage-2 paths. One-time URLs (404 after use). Client binding via `af_uuid`. CORS `*`, `no-store` cache. Payload iteration v1->v2 expanding theft scope.

**FRP Backdoor (`sys-gateway`)**: Customized FRP (`yfrp`) disguised as "System Gateway Service". Installed to `/usr/bin/sys-gateway` with init script at `/etc/init.d/sys-gateway`. Provides persistent reverse tunnel independent of Apifox process.
