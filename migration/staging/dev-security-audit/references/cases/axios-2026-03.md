# axios npm Supply Chain Attack (March 2026)

## Summary

On 2026-03-31, two malicious versions of the axios HTTP client library were published to npm: `axios@1.14.1` and `axios@0.30.4`. The attacker hijacked the npm account of the lead maintainer (`jasonsaayman`), changed the account email to `ifstap@proton.me`, and manually published via npm CLI — bypassing the legitimate GitHub Actions OIDC Trusted Publisher workflow. Both versions inject a fake dependency `plain-crypto-js@4.2.1` whose sole purpose is a `postinstall` hook that deploys a cross-platform Remote Access Trojan (RAT). The malicious versions were live for approximately **2 hours 53 minutes** before npm removed them. The RAT dropper self-deletes after execution and replaces its own `package.json` with a clean version, making post-infection `node_modules` inspection unreliable.

**Key difference from app-based cases**: axios is a library dependency (`product_type: npm-library`), not an installed application. A single machine may have multiple projects using axios. **Phase 0 presence check**: instead of checking for an installed app, scan for any `package-lock.json`/`yarn.lock`/`pnpm-lock.yaml` containing axios. If no lockfile references axios, status is `NOT_INSTALLED`.

### Self-Deletion Behavior

After payload delivery, the dropper performs 3 cleanup steps:
1. Deletes `setup.js` (the postinstall script)
2. Deletes the original `package.json` (containing postinstall hook)
3. Renames pre-staged clean `package.md` to `package.json`

Post-cleanup, `node_modules/plain-crypto-js/` appears as a normal crypto-js copy. **Lockfile diffing is more reliable than node_modules inspection.**

## Detection Commands

### Step 1: Check axios presence (lockfile scan)

Axios is a library — scan all projects on the machine for compromised versions. Uses resolved tarball URL for reliable detection (avoids multiline JSON matching issues in package-lock.json v2/v3).

**macOS / Linux:**

```bash
# package-lock.json: match resolved tarball URL (works across lockfile versions)
find "$HOME" -maxdepth 6 -name "package-lock.json" -not -path "*/node_modules/*" -print0 2>/dev/null | while IFS= read -r -d '' lockfile; do
  if grep -qE 'axios-(1\.14\.1|0\.30\.4)\.tgz' "$lockfile" 2>/dev/null; then
    echo "COMPROMISED_VERSION: $lockfile"
  fi
done

# yarn.lock: version line follows package header
find "$HOME" -maxdepth 6 -name "yarn.lock" -not -path "*/node_modules/*" -print0 2>/dev/null | while IFS= read -r -d '' lockfile; do
  if grep -qE 'axios-(1\.14\.1|0\.30\.4)\.tgz' "$lockfile" 2>/dev/null; then
    echo "COMPROMISED_VERSION: $lockfile"
  fi
done

# pnpm-lock.yaml
find "$HOME" -maxdepth 6 -name "pnpm-lock.yaml" -not -path "*/node_modules/*" -print0 2>/dev/null | while IFS= read -r -d '' lockfile; do
  if grep -qE 'axios@(1\.14\.1|0\.30\.4)' "$lockfile" 2>/dev/null; then
    echo "COMPROMISED_VERSION: $lockfile"
  fi
done
```

**Windows** (user runs manually):

```powershell
Get-ChildItem -Path $env:USERPROFILE -Recurse -Depth 6 -Include "package-lock.json","yarn.lock","pnpm-lock.yaml" -ErrorAction SilentlyContinue |
  Where-Object { $_.FullName -notmatch "node_modules" } |
  ForEach-Object { if (Select-String -Path $_.FullName -Pattern 'axios-(1\.14\.1|0\.30\.4)\.tgz|axios@(1\.14\.1|0\.30\.4)' -Quiet) { "COMPROMISED_VERSION: $($_.FullName)" } }
```

### Step 2: Check for malicious dependency

```bash
# Check all node_modules for plain-crypto-js
find "$HOME" -maxdepth 7 -path "*/node_modules/plain-crypto-js" -type d 2>/dev/null
```

**macOS:**

```bash
test -f "/Library/Caches/com.apple.act.mond" && echo "FOUND: macOS RAT artifact"
test -f "${TMPDIR:-/tmp}/6202033" && echo "FOUND: dropper temp file"
```

**Linux:**

```bash
test -f "/tmp/ld.py" && echo "FOUND: Linux RAT artifact"
```

**Windows** (user runs manually):


```bash

lsof -i -P 2>/dev/null | grep "142.11.206.73" && echo "FOUND: Active C2 connection"
```

## Interpretation Guide

| Lockfile Match | plain-crypto-js Found | RAT Artifact Found | Status | Action |
|----------------|----------------------|---------------------|--------|--------|
| Yes | Yes or No | Yes | **COMPROMISED** | Full credential rotation + forensic investigation |
| Yes | Yes or No | No | **COMPROMISED** (RAT self-deleted) | Credential rotation — RAT ran and cleaned up |
| No | Yes | Any | **COMPROMISED** (related package) | Check for `@shadanai/openclaw`, `@qqbrowser/openclaw-qbot` |
| No | No | Yes | **COMPROMISED** (lockfile cleaned) | RAT artifacts prove execution occurred |
| No | No | No (axios installed) | **INCONCLUSIVE** | Verify lockfile wasn't modified; check git history of lockfile during attack window |
| N/A | No | No (axios not installed) | **CLEAN** | No action needed |

**Note on INCONCLUSIVE**: If axios is installed but at a safe version and no artifacts found, the system is likely clean. However, if `npm install` ran during the attack window (00:21–03:15 UTC on 2026-03-31) with a caret range (`^1.14.0` or `^0.30.0`), the compromised version may have been pulled and the RAT may have self-deleted. Check `package-lock.json` git history for version changes during that window.

## Evidence Preservation

Before cleanup, archive forensic evidence:

```bash
umask 077
EVIDENCE_DIR="$(mktemp -d "$HOME/axios-evidence.XXXXXX")"
chmod 700 "$EVIDENCE_DIR"

# Lockfiles with compromised versions (tarball URL match — multiline-safe)
find "$HOME" -maxdepth 6 \( -name "package-lock.json" -o -name "yarn.lock" -o -name "pnpm-lock.yaml" \) -not -path "*/node_modules/*" -print0 2>/dev/null | while IFS= read -r -d '' lockfile; do
  if grep -qE 'axios-(1\.14\.1|0\.30\.4)\.tgz|axios@(1\.14\.1|0\.30\.4)' "$lockfile" 2>/dev/null; then
    DEST="$EVIDENCE_DIR/$(printf '%s' "$lockfile" | tr '/' '_')"
    cp "$lockfile" "$DEST"
  fi
done

# Malicious dependency (if still present — may have self-cleaned)
find "$HOME" -maxdepth 7 -path "*/node_modules/plain-crypto-js" -type d -print0 2>/dev/null | while IFS= read -r -d '' dir; do
  HASH=$(printf '%s' "$dir" | shasum -a 256 2>/dev/null | cut -c1-12 || printf '%s' "$dir" | md5sum 2>/dev/null | cut -c1-12)
  cp -r "$dir" "$EVIDENCE_DIR/plain-crypto-js-${HASH}" 2>/dev/null
done

# RAT artifacts
cp "/Library/Caches/com.apple.act.mond" "$EVIDENCE_DIR/" 2>/dev/null  # macOS
cp "/tmp/ld.py" "$EVIDENCE_DIR/" 2>/dev/null                          # Linux
cp "${TMPDIR:-/tmp}/6202033" "$EVIDENCE_DIR/" 2>/dev/null              # All

echo "Evidence archived to: $EVIDENCE_DIR"
```

## Cleanup

### Step 1: Remove compromised versions

For each affected project:

```bash
cd <project-directory>
# Remove node_modules entirely
rm -rf node_modules

# Pin safe version in package.json FIRST (before any install)
npm pkg set dependencies.axios="1.14.0" 2>/dev/null  # for 1.x users
# or: npm pkg set dependencies.axios="0.30.3"         # for 0.x users

# Reinstall with --ignore-scripts to prevent any postinstall hooks from firing
npm install --ignore-scripts

# After verifying lockfile is clean, reinstall normally to run legitimate scripts
npm ci
```

### Step 2: Remove RAT artifacts

**macOS:**

```bash
rm -f "/Library/Caches/com.apple.act.mond"
rm -f "${TMPDIR:-/tmp}/6202033"
```

**Linux:**

```bash
rm -f "/tmp/ld.py"
rm -f "${TMPDIR:-/tmp}/6202033"
```

**Windows** (user runs manually):

```cmd
del /f "%PROGRAMDATA%\wt.exe"
del /f "%TEMP%\6202033.vbs"
del /f "%TEMP%\6202033.ps1"
```

### Step 3: Credential rotation

If any detection step returned COMPROMISED, assume the RAT exfiltrated credentials. See `references/remediation.md` for full procedures. Priority:

1. npm tokens (`~/.npmrc`) — supply chain risk
2. Cloud credentials (AWS/GCP/Azure)
3. SSH keys
4. Git platform tokens (GitHub/GitLab)
5. All tokens in shell history and `.env` files

### Post-Cleanup Verification

```bash
# Verify no compromised versions remain (tarball URL match)
find "$HOME" -maxdepth 6 \( -name "package-lock.json" -o -name "yarn.lock" -o -name "pnpm-lock.yaml" \) -not -path "*/node_modules/*" -print0 2>/dev/null | \
  xargs -0 grep -lE 'axios-(1\.14\.1|0\.30\.4)\.tgz|axios@(1\.14\.1|0\.30\.4)' 2>/dev/null
# Should return nothing

# Verify no RAT artifacts
test -f "/Library/Caches/com.apple.act.mond" && echo "STILL PRESENT" || echo "CLEAN"  # macOS
test -f "/tmp/ld.py" && echo "STILL PRESENT" || echo "CLEAN"                          # Linux
```

## Appendix: Attack Chain Summary

```
Attacker compromises jasonsaayman npm account (method unknown)
  → Changes account email to ifstap@proton.me
  → Pre-stages plain-crypto-js@4.2.0 (clean decoy, 18h before)
  → Publishes plain-crypto-js@4.2.1 (malicious postinstall)
  → Publishes axios@1.14.1 (adds plain-crypto-js dependency)
  → Publishes axios@0.30.4 (39 min later, second branch)
  → npm install triggers postinstall: node setup.js
    → setup.js: 2-layer deobfuscation (reversed Base64 + XOR "OrDeR_7077")
    → Contacts C2: sfrclak.com:8000/6202033
    → Downloads platform-specific RAT:
      macOS: /Library/Caches/com.apple.act.mond (AppleScript + zsh)
      Windows: %PROGRAMDATA%\wt.exe (copied PowerShell) + .vbs launcher + .ps1
      Linux: /tmp/ld.py (nohup python3)
    → RAT capabilities: fingerprinting, 60s beacon, command exec, binary injection
    → Self-deletion: setup.js deleted, package.json replaced with clean version
  → Initial issue #10590 deleted by attacker (admin access)
  → Issue #10604 opened, npm removes versions (~2h53m exposure)
```
