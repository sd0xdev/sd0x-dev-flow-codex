## Content Rules

| Rule | Description |
|------|-------------|
| Problem-space only | No solution design, no architecture, no implementation details |
| Testable requirements | Every FR must be verifiable via test or observation |
| Prioritized | Every FR has MoSCoW priority + rationale |
| Measured NFRs | Every NFR has a quantifiable metric or threshold |
| Traced assumptions | Every assumption notes its source |
| No secrets | Never include API keys, tokens, passwords |
